dapp32

A proof of concept for rendering small web app flows from on-chain contracts, including wallet-signed and gasless transactions.

View source Open PoC

What it is

dapp32 is a generic viewer for decentralized app interfaces. Instead of shipping a separate backend and frontend for each contract, a contract exposes view functions that return JSON interface descriptions. The browser reads those descriptions, renders controls, collects values, and calls the next contract function.

The target route has the shape /ui/[network]/[address]/[view]. That gives the viewer enough information to find a chain, connect to the contract, and ask for the first screen.

How the flow works

1 Open a route

The URL names the chain, contract address, and first view function.

2 Read the ABI

The viewer calls abiURI() and loads the contract interface.

3 Fetch UI JSON

A view or pure function returns the screen definition.

4 Render controls

JSON elements become text, inputs, selects, buttons, QR codes, or NFT views.

5 Call the contract

Buttons trigger read-only calls, wallet transactions, or gasless OpenGSN relay calls.

Core pieces

Contract

Provides abiURI(), view functions, transaction functions, and optional success or failure screens.

Viewer

The Next.js app loads the ABI, prepares variables, renders UI JSON, and dispatches contract calls.

Wallet and relay

Users can sign transactions directly, while gasless actions can route through an OpenGSN paymaster.

Current status

This repository is an early proof of concept. The active path is the Next.js viewer under poc/dapp32-nx and the sample Hardhat contract under poc/on-chain.
