Makefile operator guide

The Makefile is the supported control plane for this repository. It keeps routine work Docker-backed, gives each task an explicit trust boundary, and prevents dependency installation, local chains, RPC access, and interactive debugging from blending into one broad authority.

The shape

Install deps and package-deps are the only normal networked dependency lanes.
Verify deps-verify and checks validate locked inputs, repo shape, and Compose posture.
Build build, script-build, and package-build compile without network.
Test test, fuzz, invariant, and package-test run controlled checks.
Operate anvil-*, cast-rpc, viewer-terminal, and scenario targets are explicit.

Default posture

Non-root

Docker lanes refuse root execution through NON_ROOT_GUARD. Runtime containers use the caller's UID/GID or a fixed unprivileged UID.

Offline

Ordinary check, build, test, fuzz, invariant, coverage, package build, and package test lanes run with network_mode: "none".

Read-only

Routine lanes mount the repo read-only and use tmpfs for caches, artifacts, and broadcasts. Writable lanes are named and narrow.

Cap-free

Compose services drop Linux capabilities, set no-new-privileges:true, and bound memory and process counts.

Explicit state

Long-lived services use separate Compose project names. Targets such as anvil-down and viewer-terminal-down encode cleanup.

No hidden updates

Default dependency targets install locked material only. Metadata changes require ALLOW_UPDATE=1.

Targets

Target Purpose Authority
deps Install locked Soldeer dependencies into dapps/dependencies/. Networked stage through an HTTPS allowlist proxy; offline apply writes only dependency outputs.
package-deps Install locked npm workspace dependencies into root node_modules/. Networked stage sees only temporary package manifests; offline apply writes node_modules/.
checks Run Python repository checks: hygiene, source policy, dependency metadata, and Compose posture. Offline, read-only, no Docker socket; Compose rendering only.
fmt Check Solidity formatting for dapp src, test, and script trees. Offline and read-only.
build Compile ordinary dapp Solidity source trees. Runs after dependency verification; offline and read-only.
script-build Compile deployment scripts without executing or broadcasting them. Offline and read-only; execution belongs to scenario lanes.
abi Generate ABIs from dapps/<name>/src/*.sol into existing CAM ABI directories. Offline, but intentionally writes under dapps/<name>/cam/abi/.
test Run repository checks and Forge unit tests. Verifies dependencies first, then runs checks and offline unit tests.
fuzz, invariant, coverage Run heavier Forge lanes by convention over dapp test trees. Offline and read-only after dependency verification.
package-build, package-test Build and test TypeScript workspace packages. Offline consumers of locked node_modules/; build writes only package dist/.
ci Run the default local confidence path. Composes routine formatting, compilation, contract tests, and package tests.

Dependency lanes

Soldeer

make deps stages downloads through compose/deps.yml, verifies upstream archives, applies declared patches, writes checksums, and only then copies material into dapps/dependencies/.

npm

make package-deps stages a minimal manifest tree through compose/package-deps.yml. Install scripts are disabled, registry access is allowlisted, and lock updates require ALLOW_UPDATE=1.

Boundary

Networked dependency tools do not receive repo-root write mounts. Repository writes happen later in offline apply services with narrow target paths.

Runtime and scenario lanes

Anvil

anvil-internal starts a Docker-only chain. anvil-host binds to 127.0.0.1 for host tools. The plain anvil target fails so the access boundary must be chosen explicitly.

RPC

cast-rpc reads an external RPC URL from a temporary file or explicit file path, unsets the raw environment variable, and routes through a method-filtering proxy.

Viewer

viewer-terminal is a mock-backed, networkless interactive session. Separate status, attach, and down targets make the long-running container visible and cleanable.

Bike fixture

bike-nft-local-deploy starts an internal Anvil, reads the deployer key from a Compose secret, requires CAM_URI, broadcasts the local fixture, and removes the stack on exit.

Secrets

Use file-backed secrets for private keys and RPC URLs. Do not commit, log, or paste production keys into shell history.

External access

The only intended external network lanes are dependency staging, live egress checks, and proxied RPC inspection. Everything else should stay offline unless a new Compose boundary is reviewed.

How to read changes to the Makefile

A new target should exist only when it has a distinct operator purpose or trust boundary. Routine checks should reuse existing offline services; network, host ports, secrets, writable mounts, browsers, RPC, and long-running containers should get explicit Compose services and explicit cleanup behavior.

  • If it downloads dependencies, split staging from applying.
  • If it writes the repo, make the writable path narrow and generated.
  • If it runs with secrets, prefer file-backed secrets and unset raw environment variables before Compose.
  • If it starts a service, provide a stop/status path and a separate Compose project name.
  • If it changes posture, add rendered Compose checks under tests/checks/.
